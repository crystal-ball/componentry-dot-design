# Component API Design

Every component accepts a set of library props that cause utility classes or component
styles to be rendered.

| Prop            | Value                                     |
| --------------- | ----------------------------------------- |
| `background`    | `$background-colors()`                    |
| `border`        | boolean                                   |
| `borderTop`     | boolean                                   |
| `borderRight`   | boolean                                   |
| `borderBottom`  | boolean                                   |
| `borderLeft`    | boolean                                   |
| `borderColor`   | `$border-colors()`                        |
| `borderWidth`   | sm, lg, xl                                |
| `fontColor`     | `$typography-colors()`                    |
| `fontSize`      | string, number, xs, sm, md, lg, xl        |
| `fontWeight`    | lighter, light, normal, bold, bolder      |
| `letterSpacing` | string                                    |
| `lineHeight`    | string                                    |
| `italic`        | boolean                                   |
| `monospace`     | boolean                                   |
| `uppercase`     | boolean                                   |
| `textAlign`     | left, center, right, justify              |
| `p*`            | string, number, `$spacing`                |
| `m*`            | string, number,`$spacing`                 |
| `position`      | static, relative, absolute, fixed, sticky |
| `width`         | string, number                            |
| `height`        | string, number                            |
| `minWidth`      | string, number                            |
| `minHeight`     | string, number                            |
| `maxWidth`      | string, number                            |
| `maxHeight`     | string, number                            |

## Sizes

Style props that accept sizes are configured to handle a set of library values as
classNames and pass through all other values. Setting the size to one of: `xs`, `sm`, `md`
`lg`, or `xl` will return a class, passing any other string or numeric value will return a
style.

```jsx
// Will render utility class: `.p-md`
<Block p='md' />

// Will render style: `{ padding: 5 }`
<Block p={5} />
```

## Component `variant` API

Components each have at least one `variant`, components with base styles use this to
determine the base styles for the element. You can create additional variations if you
need multiple styles of a single component. For example you may want a menu with dense
spacing with a button trigger style, as well as an additional menu for navigation with
loose spacing and a text trigger style.

- The `variant` prop will override the base css class used throughout the component.
- The component behaviors are separated into a separate class so that styles can be
  customized without having to reproduce behaviors.
